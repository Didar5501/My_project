# Описание Базы Данных

## База Данных 1 (БД1)

Этот проект будет включать в себя Базу Данных 1 (БД1) на основе СУБД MySQL, которая служит основой для хранения данных.

Структура БД1

БД1 (База данных учетной системы) включает в себя следующие таблицы:

- `action`: Эта таблица хранит информацию о действиях  по контракту (actions), выполненных в учетной системе. Эти данные представляют информацию о различных действиях с указанными атрибутами, такими как дата операции, идентификатор контракта, суммы и другие характеристики. Связана с таблицей Operation по полю OperationID

- `operation`: В данной таблице хранятся данные об операциях (operations), связанных с контрактами. Эти данные представляют информацию о различных операциях с указанными атрибутами, такими как тип операции, дата операции, идентификатор контракта, суммы и другие характеристики. Связана с таблицей action по полю OperationID (один ко многим)

- `tranche`: Эта таблица содержит информацию о траншах (tranches) в контрактах. Она включает в себя данные о версии транша, датах выплат, суммах и другие характеристики. 
Стоит упомнять что в таблице tranche нет поля ContractID. Дабы отфильтровать данные по контрактам необходимо в запрос включить таблицу Shedule по полю SheduleID. И тогда можно будет отфильтровать по полю s.ContractID

## База Данных 2 (БД2)

База Данных 2 (БД2) создана для целей аудита и мониторинга данных, хранящихся в БД1. Она включает в себя таблицы аудита для каждой из таблиц БД1, а также таблицу `procedureAud`, фиксирующую запуск процедур.

### Таблицы аудита

- `actionAudit`: В этой таблице хранятся аудиторские записи для действий из таблицы `action`. Записи включают в себя информацию о пользователе, выполнившем действие, дату и описание действия.

- `operationAudit`: Эта таблица содержит аудиторские записи для операций из таблицы `operation`. В ней сохраняется информация о пользователе, совершившем операцию, ее типе, дате и других характеристиках.

- `trancheAudit`: Таблица аудита для данных о траншах из таблицы `tranche`. Здесь содержится информация о пользователях, внесших изменения, датах и характеристиках траншей.

### Таблица `procedureAud`

- `procedureAud`: Эта таблица фиксирует запуск процедур в системе. Она связана с таблицами аудита (`actionAudit`, `trancheAudit`) по идентификаторам аудита и содержит информацию о пользователе, контракте и названии запущенной процедуры. Так же стоит упомянуть что процедура принимате один параметр ContractID - индетификатор контракта.
